Authentifizierungsmethode

Alle Nutzer/innen haben eine primäre Methode um sich gegenüber Mahara zu authentifizieren. In einigen speziellen Fällen, können sie aber einen zweiten Weg der Authentifizierung besitzen.

Hier können Sie festlegen, welche Methoden die Nutzer/innen wählen können, um sich gegenüber Mahara zu authentifizieren. In den meisten Fällen werden Sie diese Möglichkeit aber nicht brauchen, da die Optionen für Sie korrekt angelegt werden.

In eingen Fällen, wird es vielleicht erforderlich sein, dass sich die/der Nutzer/in mit einer unterschiedlichen Methode anmeldet, wie sie zuvor benutzt wurde. Zum Beispiel kann der Zugang über einen LDAP Server nicht mehr vorhanden sein, die Nutzer/innen wollen aber weiterhin auf ihren Mahara Account zugreifen, in diesem Fall können Sie eine andere Methode, wie zum Beispiel 'Internal (Keine Einrichtung)' einstellen. die Nutzer/innen können sich dann direkt bei Mahara ohne den LDAP Server anmelden - es ist aber sinnvoll, dass Sie das Feld 'Zurücksetzen des Passworts' benutzen und so eine Passwortänderung für die Nutzer/innen einleiten.
