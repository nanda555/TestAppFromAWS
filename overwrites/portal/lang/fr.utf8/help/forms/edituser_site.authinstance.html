Méthode d'authentification

Tous les utilisateurs ont une méthode d'authentification primaire pour s'identifier dans Mahara. Dans certains cas spécifiques, ils peuvent également avoir une méthode secondaire, mais la plupart du temps ils ne s'authentifient que d'une seule façon.

Ici, vous pouvez choisir le mécanisme d'authentification dans Mahara. Dans la plupart des cas vous n'aurez pas à toucher à ce réglage, car il aura déjà été spécifié correctement pour vous.

Dans certaines circonstances, vous pouvez être amenés à définir une méthode différente d'authentification pour un utilisateur. Par exemple, si l'utilisateur s'authentifiait auparavant sur le LDAP et que le serveur LDAP ne le connaît plus, vous pourriez choisir une autre méthode comme 'Interne' (Pas d'institution). L'utilisateur pourrait dans ce cas s'identifier directement dans Mahara sans passer par un serveur LDAP. Dans ce cas, vous devrez d'ailleurs choisir un mot de passe initial grâce au champ 'Réinitialiser le mot de passe'.
