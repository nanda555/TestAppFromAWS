Plugin d'authentification

Ici vous pouvez choisir comment les utilisateurs de cette institution vont s'authentifier. Chaque institution devrait avoir au moins une méthode d'authentification qui lui est attachée.

Le plugin d'authentification interne recherche les noms d'utilisateurs et mots de passe dans la base de données du site. Les utilisateurs doivent avoir un compte pour que ceci fonctionne, ce qui peut être fait en autorisant l'inscription par l'intermédiaire du formulaire d'inscription, ou utiliser les écrans Ajouter utilisateur ou Importer utilisateurs.

La méthode d'authentification réglée sur Aucune permet à n'importe qui d'entrer. Les utilisateurs peuvent choisir n'importe quel nom d'utilisateur ou mot de passe. Ceci ne doit être utilisé que pour les tests!

Les autres méthodes d'authentification reposent généralement sur des sources externes comme un serveur LDAP ou une application distante. Elles ont leurs propres paramètres que vous pouvez configurer dans une fenêtre surgissante, lorsque vous les ajoutez.

Un paramètre qu'ils partagent habituellement est de définir si les comptes peuvent être automatiquement créées dans Mahara lorsqu'un utilisateur s'inscrit pour la première fois (en opposition avec les comptes créés précédemment par l'administrateur).
